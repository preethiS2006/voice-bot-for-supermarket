================================================================================
NOVA FRESH AI VOICE BOT - DETAILED PROJECT DOCUMENTATION
================================================================================

1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
The Nova Fresh AI Voice Bot is a high-speed, hands-free browser-based voice assistant built specifically for supermarkets and grocery shoppers. It completely eliminates text input. Users simply speak to the bot in a conversational way (either English or Tamil) to query inventory, add items to an order, cancel items, and finalize a purchase. 

The application maintains a real-time, live-updating "Excel-style" order sheet in the browser right alongside an interactive voice visualizer.


2. TECH STACK
--------------------------------------------------------------------------------
This project takes a "Vanilla First" approach to maximize speed and minimize latency, using no heavy JavaScript frameworks like React, Vue, or Angular.

A. Frontend Architecture:
   * HTML5: Structural layout including glassmorphic styling grids and the live data table.
   * CSS3: Advanced styling using CSS Variables, flexbox, grid, and animations like ripple effects for the microphone.
   * JavaScript (ES6+): Pure vanilla JS for managing the application state (cart tracking, inventory mapping, DOM manipulation).

B. Voice Processing (The "Ears" and "Mouth"):
   * Web Speech API (SpeechRecognition): Handles Speech-to-Text (STT). It records user audio via the microphone and transcribes it instantly into text (English or Tamil) inside Google Chrome.
   * Web Speech API (SpeechSynthesis): Handles Text-to-Speech (TTS). It takes the text responses generated by the bot and reads them out loud using the operating system's built-in voices.

C. Environment Setup:
   * Local Web Server: The application strictly requires a local server (like XAMPP `htdocs`, Node's `live-server`, or VS Code Live Server). Browsers block SpeechRecognition access if ran via a static `file:///` path for security reasons.


3. APIs AND AI MODELS USED
--------------------------------------------------------------------------------
While the Web Speech API manages audio capture/playback, the core intelligence (understanding exactly what the user actually wants) is managed by an LLM (Large Language Model) or a fallback matching engine.

A. The Cloud AI Architecture (When Setup):
   * API Provider: Groq (via `https://api.groq.com/openai/v1/chat/completions`)
   * AI Model Used: `llama3-8b-8192` (Meta's LLaMA 3 8-Billion parameter model)
   
   Why Groq and LLaMA 3? 
   Voice bots require instant reasoning. If the AI takes 3 seconds to reply, the conversation feels unnatural and broken. Groq uses physical LPUs (Language Processing Units) that achieve incredibly fast inference speeds (sometimes >800 tokens a second). 
   Meta's LLaMA-3 (8B) is the perfect size: it's incredibly fast but smart enough to follow the strict system prompts. It's prompted to parse complex user sentences into rigid data structures like `[ADD: tomato, 2kg]` while simultaneously generating a short, human-like voice response (e.g. "Added 2kg of tomatoes, what else?").

B. The Built-In Fallback Engine (Dummy DB Logic):
   If the Groq API key is left empty, the system falls back onto a robust, hard-coded vanilla JavaScript Regex & Keyword Matching engine. This allows the bot to work offline and still handle inventory parsing, quantities, price checking, and checkout.


4. DETAILED VOICE COMMAND REFERENCE
--------------------------------------------------------------------------------
Whether utilizing the LLaMA 3 LLM or the internal fallback engine, the system targets specific words to activate cart logic. Below is exactly what words activate what actions.

I. THE INVENTORY (பொருட்கள்)
Your bot knows 6 items. You must say the name of the item to interact with it:
* Tomato -> English: 'Tomato'      | Tamil: 'தக்காளி' (Thakkali)
* Onion  -> English: 'Onion'       | Tamil: 'வெங்காயம்' (Vengayam)
* Potato -> English: 'Potato'      | Tamil: 'உருளைக்கிழங்கு' (Urulaikilangu)
* Milk   -> English: 'Milk'        | Tamil: 'பால்' (Paal)
* Apple  -> English: 'Apple'       | Tamil: 'ஆப்பிள்' (Apple)
* Grapes -> English: 'Grapes'      | Tamil: 'த்ராட்சை' (Thratchai)

II. ADDING ITEMS TO CART (பொருட்களைச் சேர்க்க)
Along with the item name, your bot listens for "Action Keywords" that indicate you want to buy.

English "Add" Words:
- 'order', 'buy', 'need', 'want', 'give', 'have', 'add'
  (Example: "Give me half a kilo of Apple.")

Tamil "Add" Words:
- 'ஆர்டர்' (Order)
- 'வேண்டும்' / 'வேணும்' (Vendum / Venum - I want)
- 'கொடுங்க' (Kodunga - Give)
- 'இருக்கா' (Iruka - Do you have)
- 'சேர்த்துவிடு' / 'போடுங்க' (Serthividu / Podunga - Add it)
  (Example: "எனக்கு இரண்டு கிலோ தக்காளி வேண்டும்")

III. REMOVING ITEMS (பொருட்களை நீக்க)
If a user changes their mind, they can subtract items.

English "Remove" Words:
- 'remove', 'cancel', 'delete', 'subtract', 'minus', "don't want"
  (Example: "I don't want the potato.")

Tamil "Remove" Words:
- 'நீக்கு' (Neeku - Remove)
- 'வேண்டாம்' / 'வேணாம்' (Vendam / Venam - I don't want)
- 'குறைக்க' / 'குறை' (Kuraikka / Kurai - Reduce)
- 'எடுத்துடு' (Eduthudu - Take it out)
- 'திரும்பப் பெறு' (Thirumbai peru - Take back)
  (Example: "ஆப்பிள் வேண்டாம்")

IV. QUANTITIES AND NUMBERS (எண்கள்)
The system listens to numeric words instead of just rigid digits. 
In English, it parses: 'one', 'two', 'half', 'quarter', etc.
In Tamil, it parses: 'ஒன்று', 'ஒரு' (oru - one), 'ஓர்', 'இரண்டு', 'ரெண்டு' (rendu - two), 'அரை' (arai - half), 'கால்' (kaal - quarter).

V. CHECKOUT AND EXPORTING DATA (பில் போட / செக் அவுட் செய்ய)
There is a specific state lock on the "Export Orders" button. This button is initially Disabled. It ONLY unlocks when the user verbally declares they are finished shopping, triggering the checkout function. At this point, the bot reads out the total sum, and the user can download their CSV.

English Checkout Phrases:
- 'checkout' or 'check out'
- 'confirm order' or 'confirm cart'
- 'done ordering' or 'finish' or 'complete my order'

Tamil Checkout Phrases:
- 'பில் போடு' / 'பில் போடுங்கள்' (Bill podu / Bill podungal)
- 'ஆர்டர் முடிந்தது' (Order mudinthathu - Order finished)
- 'முடித்துவிட்டேன்' (Mudithuvitten - I am finished)
- 'கன்பார்ம்' / 'கன்ஃபர்ம்' (Confirm)
- 'ஆர்டர் உறுதிப்படுத்து' (Order uruthipaduthu - Confirm the order)
  (Example: "பில் போடுங்கள்")
